Что такое Git?

Git — это самая популярная распределенная система управления версиями, используемая для отслеживания изменений в исходном коде программ и других файлах
Что такое репозиторий?

Репозиторий (от англ.repository — хранилище) — это место хранения цифровых данных, чаще всего исходного кода программных проектов, их истории изменений и сопутствующих файлов

Зачем нужны коммиты?

Коммиты (commit) — это зафиксированные снимки состояния вашего кода в системе контроля версий (Git), которые позволяют сохранять историю изменений, работать над проектом совместно, легко отменять ошибки и возвращаться к рабочим версиям

## Работa через Pull Request


**Изменения.** Нужно перечислить вносимые изменения списком. Если автор подробно описывал их в сообщениях коммитов, то можно взять текст из них. Если какие-то сообщения требуют дополнительного пояснения, то можно указать важные сведения в виде подпунктов. 

**Проблемы.** Если автор в чём-то сомневается или замечает противоречия, то лучше изложить их в этом разделе. Также можно упомянуть моменты, в которых бы не помешал совет от коллег. 

**Примечания.** Этот раздел служит для перечисления моментов, которые не столь очевидны. Например, можно указать, что ввели новую команду сборки, которая производит копирование файлов, или использовали какой-то функционал с «волшебными цифрами». 

**Ссылки на документы.** В этом разделе можно приложить любые соответствующие документы, которые нужно прикрепить к описанию PR. Это могут быть документы о требованиях к продукту, архитектурные диаграммы, схемы системы базы данных и другие. 

**Зависимые PR (если есть).** Этот раздел объясняет любой зависимый код, на который опирается данный PR. Он помогает понять порядок развёртывания. freecodecamp.org
Изменения конфигурации (если есть). В этом разделе нужно указать все изменения конфигурации, которые необходимо добавить к секретам перед развёртыванием PR в производство.

**Инструкции по тестированию (если применимо).** Авторы Pull Request могут включить инструкции по проведению полного тестирования своей функции. 

**Открытые вопросы.** Иногда авторы открывают Pull Request, ещё не всё понимая, или в процессе работы находят новые вопросы. В таком случае можно включить раздел для документирования этих открытых вопросов. 

**Инструкции по развёртыванию.** Иногда в описание включают этот раздел в зависимости от того, как организован процесс развёртывания команды. Чаще всего его используют для документирования новых переменных среды, которые нужно добавить
