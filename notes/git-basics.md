# Что такое Git?

Git — это самая популярная распределенная система управления версиями, используемая для отслеживания изменений в исходном коде программ и других файлах
Что такое репозиторий?

Репозиторий (от англ.repository — хранилище) — это место хранения цифровых данных, чаще всего исходного кода программных проектов, их истории изменений и сопутствующих файлов

Зачем нужны коммиты?

Коммиты (commit) — это зафиксированные снимки состояния вашего кода в системе контроля версий (Git), которые позволяют сохранять историю изменений, работать над проектом совместно, легко отменять ошибки и возвращаться к рабочим версиям

## Что такое Pull Request?

Pull Request (PR) — это запрос на внесение изменений в код, используемый в системах контроля версий, таких как GitHub, GitLab и Bitbucket. С помощью PR разработчик может предложить свои изменения для основной ветки проекта или другой ветки, дать команде возможность просмотреть изменения, обсудить их и принять или отклонить их интеграцию

Зачем нужны ветки?

Ветки в Git нужны для изоляции изменений — работы над новыми фичами, исправлениями багов или экспериментами независимо от основной кодовой базы. Это означает, что изменения, сделанные в одной ветке, не повлияют на другие ветки, пока их не объединят
